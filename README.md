# sdnd-eldia

**仕様駆動小説開発（SDND: Spec-Driven Novel Development）** の実験プロジェクト。

人間が仕様・不変ルールを設計し、AI（Claude）が執筆・品質検査・正史管理を行う——ソフトウェア開発の手法で長編ライトノベルを書く試みです。

## 目的

AI による小説生成には、以下のような既知の問題があります。

- **設定の矛盾** — 話が進むにつれて世界設定やルールが崩れる
- **伏線の放置** — 張った伏線が回収されないまま忘れ去られる
- **キャラクターのブレ** — 口調・性格・能力が話ごとに変わる
- **品質のばらつき** — 出力のクオリティが安定しない

本プロジェクトは、これらの問題をソフトウェア工学のプラクティスで解決できるかを検証します。

## 手法

| 概念 | ソフトウェア開発 | SDND での対応 |
|------|-----------------|--------------|
| 要件定義 | 仕様書 | `specs/` — 世界設定・キャラ・魔法体系・不変ルール |
| 実装 | コーディング | `story/` — AI（writer エージェント）がエピソード本文を執筆 |
| テスト | 自動テスト / QA | `qa/` — AI（qa エージェント）が不変ルール違反・矛盾を検査 |
| リリース | デプロイ | `canon/` — 検査を通過した事実のみを正史（Canon）として確定 |
| バージョン管理 | Git | 全ファイルを Git 管理し、変更履歴を完全に追跡 |

### ワークフロー

```
人間（PM）: 仕様策定・プロンプト設計・品質判断
    ↓
writer エージェント → エピソード執筆
    ↓
qa エージェント → QA レポート作成
    ↓
  PASS? ──NO──→ writer で修正 → 再 QA
    │
   YES
    ↓
editor エージェント → Canon 登録・仕様更新
    ↓
Git commit & push
```

## 作品について

**ジャンル**: 異世界転生ファンタジー

組み込みエンジニアの桐島晴人は、残業中に意識を失い、剣と魔法の異世界に五歳児「アル＝ラグナ」として転生した。アルの目に映る魔法には「構造」があった——波形、周波数、位相。そして気づく。この世界の魔法体系には「バグ」がある。

元エンジニアが異世界の「仕様バグ」に挑む、ロジカル・ファンタジー。

## ディレクトリ構成

```
sdnd-eldia/
├── story/          # エピソード本文
│   └── arc1/       # 第1章（EP-001〜EP-008）
├── specs/          # 仕様書
│   ├── core/       # TIER 1: 不変ルール・キャラ現在状態（常時参照）
│   └── reference/  # TIER 3: 世界設定・魔法体系・キャラ詳細（必要時参照）
├── canon/          # 正史（Canon）管理
│   ├── quick_ref.md  # 最新状態スナップショット
│   ├── active/     # 直近5話の Canon エントリ
│   └── archive/    # 過去の Canon エントリ
├── qa/             # QA レポート
│   └── reports/
├── meta/           # メタ情報（伏線管理・パフォーマンスログ）
├── policies/       # 運用ポリシー（Canon承認・Git運用）
├── templates/      # テンプレート
├── tools/          # ツール（perf記録スクリプト等）
├── docs/           # ドキュメント（振り返り・プロット概要）
└── .claude/        # Claude Code エージェント設定
    └── agents/     # サブエージェント定義（writer / qa / editor）
```

## AI 利用について

本作品の本文は **ほぼ100% AI（Claude Opus）によって生成** されています。

人間が担当しているのは以下の部分です：
- 仕様書・不変ルール・プロット概要の設計
- プロンプト設計とエージェント定義
- QA 結果に基づく品質判断（PASS / 修正指示）
- ワークフロー全体のマネジメント
