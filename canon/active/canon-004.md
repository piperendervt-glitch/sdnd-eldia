# CANON-004: EP-004「最初のバグ」正史記録

<!-- 確定日: 2026-03-01 -->

## サマリー

初夏、エルダの森の縁でマナスライムが数十体異常発生した。アルは禁止を破って単独で現場に向かい、初めて意識的にデバッグスキルを発動。マナのフィードバックループ（オーバーフローバグ）の構造を視覚化し、プログラム的フォーマットのデバッグ出力を確認したが、5歳児の技量では修正できなかった。

## 確定事実

### イベント

- EVT-004-01: エルダの森の縁でマナスライムが数十体（目測30体以上）異常発生。通常は単体で存在するはずの個体が、マナのフィードバックループにより増殖し続けている。村はリンデン町のギルドに冒険者派遣を要請した。

- EVT-004-02: アルが禁止命令を破り、単独で森の縁まで向かった。大人の目を潜り抜け、草道を通って現場に接近。その後、父ドルクに発見されて帰宅を命じられた。

- EVT-004-03: アルが初めて意識的にデバッグスキルを発動した。EP-003の「不随意発現」と異なり、自分の意志で「見ようとする」集中行為によって発動。発動まで時間がかかった（約20〜30秒の集中）が成功した。

- EVT-004-04: デバッグスキルがマナスライム群の構造を可視化した。可視化内容:
  (1) マナの流れがフィードバックループを形成（出力が入力に帰還）
  (2) ループのゲインが2.17（unstable）
  (3) RULE-M07「オーバーフロー」バグに相当する異常

- EVT-004-05: デバッグ出力として以下のプログラム的テキストが視界に表示された。
  ```
  overflow: mana_density exceeds threshold
    at zone[elder_forest_edge:0x3F]
    loop_gain = 2.17 (unstable)
  ```
  ```
  NullRef: target_entity = undefined
    spawner logic expects valid target
    current target: null
  ```
  これはH1（バグの種類名がコンピュータ用語）の確定した初出。

- EVT-004-06: アルはバグの構造と修正の方針を理解した（フィードバックループの遮断、ヌル参照の修正）が、5歳児のマナ操作技量が足りず実行不可能と判断した。

- EVT-004-07: ドルクがアルを発見し叱責。アルに怪我はなかった。「怒っている」「でも怪我がなくて良かった」の両方を伝えた。

- このバグは未解決のまま次話（EP-005）に持ち越される。EP-005でフォレストウルフが村に降りてくる原因となる。

## 備考

- QA Minor-02（RULE-M08 追加コスト根拠補足）: 追加コスト15MPの根拠 — マナスライム30体超・フィードバックループ構造・2種類のエラー出力。中程度の複雑度として5〜50MPの範囲中央付近の15MPを適用。今後の参照基準として本エントリに記録。
- QA Warn-02（INV-D01 視認条件の解釈拡張）: デバッグスキルの発動条件として、魔法使いによる魔法行使の視認に加え、魔法的現象（マナを含む現象）の視認でも発動可能であることがEP-004で確立された。今後のデバッグスキル発動シーンでマナ系現象全般がINV-D01の視認条件を満たすという解釈を維持すること。
- QA Warn-01（マーラの定義口調の不使用）引き継ぎ: 次エピソードでマーラが子供に向けて話しかける場面では「〜だよ」「〜かしら」の語尾を意識的に使用すること。
- QA Minor-01（前世知識適用の失敗描写の重要性）引き継ぎ: L252-263の「失敗描写」が前世知識は「ヒント」に過ぎないことを示す重要な機能を持つ。次エピソード以降で削除・弱化しないこと。
- 意識的発動パターン確立（EP-005以降の参照基準）: 基礎コスト15MP + 追加コスト5〜50MP。発動手順: (1)呼吸を整えて集中、(2)解析対象を明確に意識、(3)「計測を開始する」という意思的行為、(4)20〜30秒の集中時間（慣れれば短縮の可能性）。
- H1（バグの種類名がコンピュータ用語）確定初出: EP-004。SF的真相（世界がプログラムである）はEP-029まで読者にも開示しない。
- マナスライム異常発生の解決タイミング: EP-004終了時点で未解決。EP-005でフォレストウルフが降りてくる直接原因。バグ自体の解決はEP-006（アル自身の再挑戦）の予定。
