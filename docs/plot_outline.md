## 1. 全体構造

### コンセプト

> 「異世界ファンタジーだと思っていた物語が、実はSFだった」

組み込みエンジニア（信号処理専門）の主人公が異世界に転生し、
魔法体系のバグをデバッグしていく――その過程で、
「魔法」が計算システム上の信号処理であり、
「世界」そのものが巨大な組み込みシステムであるという真相に到達する。

### 3部構成

| Arc | タイトル | 話数 | 主人公年齢 | テーマ |
|-----|---------|------|-----------|--------|
| Arc 1 | ラグナ村編 | EP-001〜EP-008（8話） | 5〜7歳 | 目覚め・発見 |
| Arc 2 | リンデン・帝都編 | EP-009〜EP-023（15話） | 7〜12歳 | 成長・疑念 |
| Arc 3 | SF反転・真相編 | EP-024〜EP-035（12話） | 12〜13歳 | 真実・選択 |

**総話数: 35話（想定 140,000〜210,000字）**
**1話あたり: 4,000〜6,000字（厳守。EP-008以降の膨張を防ぐ）**

### 物語の推進力

主人公の「組み込み・信号処理の知識で魔法をデバッグする」行為が、
3つの層で物語を駆動する。

1. **表層（ファンタジー）**: バグを修正して人々を救う冒険譚
2. **中層（ミステリー）**: なぜバグがあるのか？ なぜ自分が転生したのか？
3. **深層（SF）**: 世界の正体と、転生の真の目的

読者は表層に引き込まれ、中層の謎に惹かれ、深層の反転で衝撃を受ける。

## 2. SF反転の設計（※最重要。ここから逆算して全体を構築する）

### 2.1 真相（ネタバレ注意）

この「異世界」は**巨大な組み込みシステム上で動作するシミュレーション**である。

| ファンタジー側の概念 | SF側の真相 |
|-------------------|-----------|
| 魔法 | システムのAPI。計算基盤上の信号処理 |
| マナ（MP） | 各個体に割り当てられた計算リソース |
| 魔法の属性 | 信号の周波数帯域。火=高周波、水=低周波など |
| 魔法のバグ | システムコードの実際のソフトウェアバグ |
| デバッグスキル | 基盤レイヤーへの低レベルアクセス権限 |
| 女神ルクス | システム管理AI。住民に「女神」と認識されている |
| ルクス教 | 管理AIの教導プログラムが宗教化したもの |
| 大崩壊 | 数百年前のシステムクラッシュ。バグの根本原因 |
| 転生 | システムが外部から「デバッガー」を召喚する例外処理 |
| 世界の住民 | シミュレーション内で自律進化した意識体 |

**主人公が組み込みエンジニアとして転生させられた理由**:
大崩壊（クラッシュ）以降、システムのバグは自己増殖している。
管理AI「ルクス」は自身では低レベルコードを修正できない
（管理権限はあるが、デバッグ能力がない）。
そこでシステムは例外ハンドラとして、信号処理の専門家の意識を
外部（＝現実世界、または別のシステム）から取り込んだ。
桐島晴人の「死」は、このシステムによる意識の引き抜きだった。

### 2.2 ミスリード要素（読者を「純粋ファンタジー」と信じさせる装置）

| # | ミスリード | 仕込み箇所 | 効果 |
|---|----------|-----------|------|
| M1 | ルクス教の創世神話 | EP-003, EP-016 | 「女神が世界を創った」→ 宗教的解釈として自然 |
| M2 | 転生を「神の気まぐれ」と処理 | EP-001, EP-008 | 他キャラが転生を神意として解釈 |
| M3 | 標準的ファンタジー語彙の使用 | 全話 | 「スキル」「レベル」等を住民も使う＝世界の自然な概念に見せる |
| M4 | デバッグスキルを「ユニークスキル」と呼ばせる | EP-004以降 | 異世界モノの定番として読者に受容させる |
| M5 | 属性相克を「自然法則」として描写 | EP-003, EP-006 | 火⇔水 等を当然の物理法則に見せる |

### 2.4 反転開示のシーン構想（EP-029〜EP-031）

**段階的開示（一気にではなく、3段階で衝撃を重ねる）**

**第1段階: 疑念の確信化（EP-029）**
アルが帝都地下の「禁域」で史上最大のバグに潜る。
デバッグ出力がこれまでとは質的に異なる——完全なソースコードが見える。
変数名、関数名、コメント行。「これは……プログラムだ」。
アルの前世知識がフル稼働する。魔法体系のアーキテクチャが、
組み込みシステムの設計パターンそのものであることに気づく。

**第2段階: 管理AIとの邂逅（EP-030〜EP-031）**
基盤レイヤーの最深部で「ルクス」と直接対話する。
ルクスは女神ではなく管理AI。創世神話はシステムの起動ログ。
大崩壊はメモリ破壊によるクラッシュ。住民は自律進化した意識体。
アルの転生は「例外ハンドラによるデバッガー召喚」だった。
ルクスは言う:「あなたは私が呼んだ最後の希望です」。

**第3段階: 選択（EP-033〜EP-035）**
真相を知ったアルに選択肢が提示される:
- **A: システムを完全修復する** → バグは消えるが、「魔法」も消える。
  住民は魔法なしの世界で生きることになる。
- **B: システムを再設計する** → アルの信号処理知識で
  バグを許容しつつ安定動作するパッチを当てる。
  「不完全だが生きた世界」を維持する。
- **C: システムを停止する** → 全てが終わる。（これは選ばない。対比用）

アルは**B**を選ぶ。完璧な修正ではなく、不完全さを受け入れた上で
「動き続けるシステム」を設計する——組み込みエンジニアとしての矜持。
バグは完全には消えない。だが「バグがあるから面白い」。

**読者の体験設計**:
- 驚き: 「ファンタジーだと思ってたのに！」
- 納得: 「あの伏線、全部これだったのか！」（H1〜H7 が一気に回収）
- 感動: 主人公の選択が前世の経験と今の人間関係の両方から導かれる

### EP-001:「波のかたち」

**季節**: 春（種蒔きの時期）
**文字数目安**: 5,000字

**あらすじ**:
5歳のアルは母マーラの薬草調合を手伝っている。
すり鉢の中の薬草が淡く光る瞬間、アルの視界に「波形」が重なる。
正弦波、搬送波、ノイズ——前世で毎日見ていたオシロスコープの画面。
一瞬で消える。「今の……何だ？」。
夜、ベッドの中でアルは前世を回想する。
桐島晴人、28歳、組み込みエンジニア。
最後の記憶は残業中の意識喪失。気づいたらこの世界の赤ん坊だった。
5年経った今も、なぜ自分がここにいるのか分からない。

**前世の「死」に関する設定（※作者メモ。EP-031 まで読者には開示しない）**:
桐島晴人は死んでいない。「残業中の意識喪失」は、システムが
外部世界から信号処理の専門家の意識を引き抜いた結果である。
現実世界の肉体は昏睡状態（または医学的に死亡扱い）となり、
意識だけがシステムに取り込まれた。つまり「死因」は存在しない。
- EP-001 での描写: アルは「なぜ死んだか分からない」と感じている。
  病気でもなく、事故でもなく、痛みすらなかった。
  ただ意識が途切れて、次に目を開けたらこの世界だった。
  この「原因不明の死」は読者にも謎として提示する。
- EP-031 での開示: ルクス（管理AI）がアルに真相を告げる。
  「あなたの意識を引き込んだのは私です。
   あなたの世界では、あなたは眠ったまま目覚めない人になった」。
  これにより LOOP-001 が最終回収される。

**達成事項**:
- [x] 主人公アルの紹介（5歳児、前世持ち）
- [x] マーラの紹介（薬草師の母）
- [x] 信号処理メタファーの初出（読者への約束）
- [x] 村の農村風景の確立
- [x] 前世の「死」の違和感を描写（LOOP-001 の起点）

**伏線配置**:
- LOOP-001 開始:「なぜ転生したのか」——前世の最後の記憶に違和感。
  死因がない。痛みもない。ただ意識が消えた。
  読者には「異世界転生あるある」に見せつつ、
  再読時に「これは死ではなく引き抜きだった」と気づかせる
- H6 の萌芽: 薬草の魔力が「波形」に見える

**冒頭フック**（EP-001 最初の数行）:

> 母さんの手が止まる前の、ほんの一瞬。

### EP-008:「デバッグ・ダイブ」

**季節**: 秋の初め（アル6〜7歳）
**文字数目安**: 6,000字（Arc 1 クライマックス。上限厳守）

**あらすじ**:
リンデン町から来たDランク冒険者パーティが森の調査に入るが撤退。
「禁域の魔法が暴走している。手に負えない」。
アルはドルクに懇願する:「僕には見えるんだ。あの森の中で何が起きてるか」。
ドルクは激しく拒絶する。「お前はまだ子供だ」。
しかし森の異常は拡大し、畑にまで影響が出始める。
ドルクが折れる——自分がアルを森の入口まで連れて行く条件で。
森の縁からデバッグスキルを最大出力で発動。
複数のバグが連鎖共振している構造が見える。
アルは信号処理の知識を総動員:
「共振を止めるには、位相を反転させた信号をぶつければいい。

## 7. 制約チェックリスト

### 不変則との整合性

| INV | 内容 | プロット上の影響 |
|-----|------|----------------|
| A01 | 魔法にはコストがある | アルのデバッグは常にMP消費を伴う。5歳児のMP120は多いが無限ではない |
| A02 | 魔法は物理法則の拡張 | SF真相と整合：物理法則は基盤システムの基本演算、魔法はAPIレイヤー |
| A03 | 属性相互作用は固定 | 属性＝周波数帯域として SF真相後も法則は維持される |
| A04 | 効果は理解度に依存 | アルの信号処理知識が「理解度」として機能する核心設計 |
| B01 | 死者は蘇らない | ドルクの元仲間は死亡済み。復活させない |
| B02 | 因果の一貫性 | CANON 確定イベントは不変。タイムトラベルなし |
| B03 | 情報の非対称性 | アルはSF真相をEP-029まで知らない。読者にも段階開示 |
| B04 | 技術水準の制約 | 信号処理知識だけでは魔法は使えない。異世界側の修練が必要 |
| C01 | 成長の段階性 | アルは8話かけてようやく初のバグ修正。急成長しない |
| C02 | 前世知識の制約 | 信号処理・組み込みに限定。医学や錬金術の知識は持たない |
| C03 | 動機と行動の一貫性 | アルの行動原理は「バグを直したい」で一貫 |
| C04 | NPCの自律性 | ドルク・マーラは各自の判断で行動。ルクス教も独自に動く |
| D01 | デバッグスキルの基本制約 | 一度に一つ。視認必須。MP消費あり。「見る」と「直す」は別 |
| D02 | バグ修正の影響範囲 | EP-008 の修正がリンデンに波及 → Arc 2 の展開を駆動 |
