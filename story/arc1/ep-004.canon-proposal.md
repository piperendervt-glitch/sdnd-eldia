## サマリー

初夏、エルダの森の縁でマナスライムが数十体異常発生した。
アルは禁止を破って単独で現場に向かい、初めて意識的にデバッグスキルを発動。
マナのフィードバックループ（オーバーフローバグ）の構造を視覚化し、
プログラム的なフォーマットのデバッグ出力を確認したが、
5歳児の技量では修正できなかった。

## 確定事実

### イベント

- EVT-004-01: エルダの森の縁でマナスライムが数十体（目測30体以上）異常発生。
  通常は単体で存在するはずの個体が、マナのフィードバックループにより増殖し続けている。
  村はリンデン町のギルドに冒険者派遣を要請した。

- EVT-004-02: アルが禁止命令を破り、単独で森の縁まで向かった。
  大人の目を潜り抜け、草道を通って現場に接近。
  その後、父ドルクに発見されて帰宅を命じられた。

- EVT-004-03: アルが初めて意識的にデバッグスキルを発動した。
  EP-003の「不随意発現」と異なり、自分の意志で「見ようとする」集中行為によって発動。
  発動まで時間がかかった（約20〜30秒の集中）が成功した。

- EVT-004-04: デバッグスキルがマナスライム群の構造を可視化した。
  可視化内容:
  (1) マナの流れがフィードバックループを形成（出力が入力に帰還）
  (2) ループのゲインが2.17（unstable）
  (3) RULE-M07「オーバーフロー」バグに相当する異常

- EVT-004-05: デバッグ出力として以下のプログラム的テキストが視界に表示された。
  ```
  overflow: mana_density exceeds threshold
    at zone[elder_forest_edge:0x3F]
    loop_gain = 2.17 (unstable)
  ```
  ```
  NullRef: target_entity = undefined
    spawner logic expects valid target
    current target: null
  ```
  これはH1（バグの種類名がコンピュータ用語）の確定した初出。

- EVT-004-06: アルはバグの構造と修正の方針を理解した（フィードバックループの遮断、
  ヌル参照の修正）が、5歳児のマナ操作技量が足りず実行不可能と判断した。

- EVT-004-07: ドルクがアルを発見し叱責。アルに怪我はなかった。
  「怒っている」「でも怪我がなくて良かった」の両方を伝えた。

## 備考

### 意識的デバッグ発動の設計メモ（次話以降のための参照用）

本エピソードで「意識的発動」が確立した。EP-005以降の発動コストは:
- 基礎コスト: 15MP
- 追加コスト: 対象の複雑さに応じて5〜50MP
- 合計: 単純な魔法現象で20MP、マナスライム群（中程度）で30MP

意識的発動に必要な手順（EP-004で確立された描写パターン）:
1. 呼吸を整えて集中する
2. 解析対象を明確に意識する
3. 「計測を開始する」という意思的行為
4. 20〜30秒の集中時間が必要（慣れれば短縮される可能性）

### H1（バグの種類名がコンピュータ用語）確定メモ

EP-004でH1が初出として確定。
- 「overflow」「NullRef」が実際のデバッグ出力として表示された
- 読者にとっては「主人公の前世知識が見え方に影響しているのかも」と解釈可能
- SF的真相（世界がプログラムである）はEP-029まで読者にも開示しない

### マナスライム異常発生の解決タイミング

このバグはEP-004終了時点で未解決。
EP-005でフォレストウルフが降りてくる直接原因となる。
バグ自体の解決はEP-006（アル自身の再挑戦）の予定。
エディタへの確認事項: EP-005のプロット設計と整合しているか確認を推奨。
