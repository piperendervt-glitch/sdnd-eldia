## 前回までのあらすじ

クラウスとの面会で変数名体系が「設計されたアーキテクチャ」として整合するという学術的補完を得た。その後、封印区画への3回目入室（俯瞰モード）を実施し、「封印区画はfoundation_layerへの穴を蓋している」という認識が確立した。integrity 31%（低下加速）の警告と入室頻度制限の勧告を受け、次の探索へ慎重に臨む姿勢を固めた。

## 本文

【第3章 第4節：逆コンパイル】

　翌々日の昼過ぎ、クラウスから連絡があった。

　「明日の午前中、学院の第二実習室を使う。来なさい」——それだけだった。

　何をするかは書かれていなかった。アルは「了解しました」と書いて返した。

　封印区画への次の入室については、今は考えていなかった。integrity の低下が加速しているという警告と、クラウスの「入室頻度を制限すること」という言葉が、そのまま今の指針だった。前回で「どこを見るべきか」の輪郭は出た。次に踏み込む前に、今持っている情報で考えられることを先にやりつくす——その判断は、二日前に固まっていた。

　だから今日のクラウスからの呼び出しは、別の何かだと思った。

　　　　　　　　　　　　＊

　第二実習室は、学院棟の西端にある広い部屋だった。通常の実習で使う部屋より天井が高く、中央に広い空間があった。アルが着いた時、クラウスはすでに部屋の端に椅子を置いて座っていた。

　部屋の中央に、四人の学院生がいた。アルが見たことのある顔もあった。属性は四つ——火、水、土、光。それぞれが杖を持ち、互いに少し距離を空けて立っていた。

「来たか」クラウスが言った。「今日は観察をしてもらう」

「何を観察しますか」

「四つの属性の同時発動だ。実習課題として彼らにやらせている。お前は発動のプロセスを——何を見るべきか分かるだろう」

　アルは少し間を置いた。「申請は」

「この部屋での発動観察は、研修プログラムの通常枠内で認められている。私が許可した範囲の発動として記録する」クラウスは少し低い声で言った。「ただし、発動はお前の判断と消耗の範囲で行いなさい。過度になりそうなら自分で止めること」

「分かりました」

　クラウスは学院生たちに向かって言った。「準備が整ったら始めなさい。四属性の順次展開——まず各自で、それから同時発動の順序で」

　学院生たちが頷いた。

　アルは部屋の端、クラウスの隣の椅子に腰を下ろした。

　　　　　　　　　　　　＊

　最初は、各自による単独発動だった。

　火属性の学院生が杖を構えた。静止した。

　その「静止」を、アルは見た。

　（前にも見た。この間だ）

　リンデンの中央広場で、荷運びの男が身体強化魔法を発動する前の間。学院の中庭で、実習中の学院生が杖を構えた時の間。今も同じものが、そこにあった。

　術者が静止している。外から見れば何も起きていない。しかし——アルの感覚に、何かが引っかかっていた。

　（申請した枠内だ。発動していい）

　アルは発動した。

　接続が来た。基礎コスト分の消耗が体温の上昇として来た。学院の実習室の空気が、少し遠くなる感触があった。

　出力が流れ始めた。

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
[SCAN: magic_interference_analysis — multi-caster / sequential]
  target: caster[fire_attr] — pre-cast phase observation
  access_level: surface_scan

  phase sequence detected:
    phase[0] — initialization
      status: ACTIVE
      duration: approx. 0.8s
      mana_state: intake / channel_align
      signature: system_call_header [attribute: fire, intent: construct]

    phase[1] — self-check
      status: pending
      trigger: phase[0] complete

    phase[2] — resource_lock
      status: pending
      trigger: phase[1] complete

    phase[3] — execute
      status: pending
      trigger: phase[2] complete

  note: phase[0] precedes caster-subjective "recognition" stage
        — external observation window: narrow (approx. 0.8s)
        — not accessible by standard analysis methods
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

　アルは出力を見た。

　（`phase[0]`——初期化フェーズ。`system_call_header`）

　「呼び出し手順のヘッダ」に相当するような語が、発動前の段階に付いていた。

　前世で「ヘッダ」と呼んでいたものは、呼び出しの前に必ず送られる定形の情報だった——どこへ向けた呼び出しか、何を要求するか、どのフォーマットで答えを期待するかを宣言する冒頭の部分。

　（魔法を発動する前の段階に——定形の情報が送られている）

　それは「認識→構築→発動」の三段階より前にある処理だった。

　火属性の学院生が発動した。炎が球状に形成され、制御された出力として前方に放たれた。

　アルは発動を止めずに、次の術者に切り替えた。水属性。同じように静止の「間」があり、同じように `phase[0]` が観測された。ヘッダに含まれる属性情報だけが違った——`attribute: water`。

　（同じ構造だ。属性が違っても、初期化の手順は同じ）

　土属性。光属性。それぞれで同じことを確認した。

　出力の形はそれぞれで同じだった。`phase[0]` — `phase[1]` — `phase[2]` — `phase[3]` の順序。`system_call_header`の存在。属性情報と意図の情報が `phase[0]` に含まれている。

　アルは発動を止めた。

　体温がわずかに高かった。額の奥が重くなり始めていた。

　（続ける。同時発動が残っている）

　クラウスが「次に進みなさい」と学院生たちに言った。

　　　　　　　　　　　　＊

　四人が同時に構えた。

　アルは発動を再開した。今度は四人を同時に視野に入れた——同時発動という一つの現象として、その構造を観察する。一人ずつではなく、四つの「静止の間」が同時に起きていた。

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
[SCAN: magic_interference_analysis — multi-caster / simultaneous]
  targets: 4 casters (fire / water / earth / light)

  phase[0] — initialization (all casters):
    caster[fire]:   phase[0] start t=0.000s
    caster[water]:  phase[0] start t=0.031s
    caster[earth]:  phase[0] start t=0.047s
    caster[light]:  phase[0] start t=0.052s

    overlap detected: phase[0] active simultaneously across 4 casters
    note: phase[0] sequences are independent — no shared channel
          each caster initiates own call_sequence

  system-layer observation:
    each phase[0] carries: [attribute] + [intent] + [caster_id]
    routing: received by mana_routing substrate
    response: channel allocation returned per caster (phase[1] trigger)

  pattern:
    this sequence matches known call_sequence protocol structure
    phase[0] functions as: request initialization / parameter declaration
    phase[1]-[3]: standard execution pipeline
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

　アルは止めた。

　膝は折れなかった。しかし足の裏に重さが来ていた。頭の後ろが締まった。

「止めます」と声に出した。

　クラウスが「分かった」と言った。

　アルは目を閉じて、一度深く息を吸った。

　出力の最後の部分が、まだ頭の中で繰り返されていた。

　`each phase[0] carries: [attribute] + [intent] + [caster_id]`。

　`routing: received by mana_routing substrate`。

　`response: channel allocation returned per caster`。

　（`mana_routing`が——応答している）

　各術者の `phase[0]` は、`mana_routing` に対して何かを「送り」——`mana_routing` が応答している。チャンネルを確保して、返してくる。

　その往復が、「静止の間」だった。

　術者が「何も動いていない」ように見えたあの時間は、実際には呼び出しの初期化が処理されていた時間だった。

　（呼び出しを受け取る側がある。`mana_routing`が、その窓口に当たっている）

　前世で組み込みシステムを扱っていた頃、割り込み処理の呼び出し手順を書いたことがあった——プロセスが動作中にシステムに対して「今からリソースを使う、受け付けてくれ」と送る初期の宣言。受け付ける側が確認を返してきて初めて、処理が始まる。

　あの往復が——魔法の発動前の「間」と同じ構造だった。

　（「呼び出し手順」がある。発動前の段階に、呼び出しを受け付ける手順が定められている）

　それを、今日確認した。

　「プログラムだ」とは言えない。「システムコールだ」とも言えない。しかし——

　（発動前に、定められた手順で何かに呼びかけて、何かが応答している。その構造が、確かにある）

　アルはゆっくり立ち上がった。

　　　　　　　　　　　　＊

　学院生たちが退室した後、クラウスが「今日見えたことを話しなさい」と言った。

　アルは少し考えてから話し始めた。

「魔法の発動前に——術者が体感する『認識・構築・発動』の三段階より前に、もう一段階あります」

「前に報告した`phase[0]`の件か」

「はい。ただ今日は、それがどんな手順を踏んでいるかを、もう少し詳しく確認できました」

　アルは言葉を選んだ。

「初期化の段階で——属性と意図と、術者を特定する情報が、`mana_routing`へ送られています。それに対して`mana_routing`が応答して、その術者のためのチャンネルを確保する。その往復が終わって、初めて次の段階に移る」

「……`mana_routing`が応答している」クラウスは静かに繰り返した。

「出力はそのように見えました。正確な解釈かどうかの確信はありません。ただ——四人の術者それぞれで同じ構造が観測されました。属性が違っても、手順は同じ。`mana_routing`との往復がある」

　クラウスは少しの間、黙った。

「術者の側からは——発動前の段階で何かと接続しているという感覚はあるか、と問えば、多くの術者は『マナが整う感覚』と言う。あるいは『何かが応じてくれる感覚』と言う者もいる」

「それが——」アルは言葉を止めた。

「術者が感覚的に把握しているものを、お前が構造として見えた、ということになるかもしれない」

「……そう解釈できるかもしれない」

　クラウスは羊皮紙を取り出した。「記録を取る。お前が見た手順の順序を、もう一度言いなさい」

　アルは話した。`phase[0]`から`phase[3]`までの順序。各フェーズのトリガー条件。同時発動での独立性。`system_call_header`という語の出現。`mana_routing`の応答。

　クラウスは書きながら聞いた。

「`system_call_header`という語について——」クラウスは書く手を止めずに言った。「それが何を意味するか、お前はどう解釈しているか」

「定められた呼び出しの宣言——どこへ向けた呼び出しか、何を必要としているかを最初に告げる手順、として見えます」アルは言った。「前世の——組み込みシステムの仕事での経験から来た解釈です。この世界でその語が全く同じ意味を持つかどうかは、確認の方法がない」

「留保付きの解釈として記録する」クラウスは言った。「それが適切だ」

　アルは頷いた。

　（`mana_routing`が受け付けている——つまり、`mana_routing`は発動の初期化の段階で外からの呼び出しを受けとる窓口として機能している）

　それは——封印区画の出力で確認してきた構造の話と、繋がっていた。

　`mana_routing`は`world_management_system`のサブモジュールとして確認されていた。`foundation_layer`の上に乗る構造の一部だった。そして今日——`mana_routing`が魔法発動の初期化において応答する主体として動いていることを確認した。

　（`mana_routing`は、呼び出しを受け付けて、チャンネルを割り当てて——それが魔法の発動を成立させている）

　それは、出力の語を「読む」という行為と、実際に魔法が発動する場面を「観察する」という行為が、初めて一つに繋がった瞬間だった。

　　　　　　　　　　　　＊

　宿への帰り道は、フィンとリーラの二人と一緒だった。二人は実習室の扉の前で待っていた。

「顔色が悪いですね」フィンが言った。

「発動の消耗です。歩けるので問題ない」

「何を見ていたんですか、今日」

　アルは歩きながら話した。魔法発動前の初期化段階。四人の術者でそれぞれ同じ構造が見えたこと。`mana_routing`が応答している可能性のこと。

　フィンは聞きながら、少しずつ考えていた。

「……発動前に`mana_routing`との往復がある、というのは——」フィンは言い方を選んだ。「つまり魔法は、術者が一人で完結させているわけじゃない、ということですか」

「そう解釈できます。ただ——」

「でも術者の感覚としては、自分が発動しているという感覚しかない」フィンは続けた。「私自身も、発動前に何かと接続している、と感じることがあります。でもそれが`mana_routing`と往復しているとは、考えたことがなかった」

「外から観測できない段階だから、理論化されていない——クラウス先生が言っていた話です」

「外から見えなかったものが、アルさんには見えた」

「今日は見えました」

　しばらくの間、三人は黙って歩いた。石畳の上に昼過ぎの光が降りていた。

　リーラが口を開いた。「つまり、魔法を使うってのは——その`mana_routing`ってやつに頼んでから使う、ってことか」

「……今日見えた範囲では、そのような構造があると解釈しています」

「頼む手順が決まっていて、その手順通りに動かないと魔法が使えない」

「はい」

「それが、バグがある場合はどうなるんだ」

　アルは少し歩きながら考えた。「`mana_routing`に問題があれば——チャンネルの確保が正常に行われない。あるいは確保されても、その後の処理に異常が混入する。リンデンで見た、属性チャンネルの劣化はそういうことだったと思っています」

「だから魔法が弱くなったり、変な動きをしたりするんだな」

「そうです」

「じゃあ——基盤の層の方にひびがある、って話はどう繋がるんだ」

　アルは少し考えてから言った。「`mana_routing`は`foundation_layer`の上に乗っています。基盤の層が壊れていれば——`mana_routing`自体も、本来あるべき状態ではないかもしれない。今の`mana_routing`は大崩壊の後に再構築されたバージョン、という出力がありました。元々あったものではない可能性がある」

「再構築された、ということは——元のやつが壊れた」

「崩壊で、壊れたか——あるいは大きく変わった。そのあとに作り直した、という解釈です。ただ作り直したとしても、その下の基盤層が壊れたままなら——根の部分の問題は解消しない」

　リーラは少し黙った。「……繋がってるな、全部」

「繋がっています」

　内環の通りに人が増え始めていた。馬車が向こうから来て、三人は端に寄った。

　フィンが歩きながら言った。「今日の観察で——アルさんが以前から気にしていた『起動手順に似ている』という感触は、答えが出ましたか」

　アルは少し間を置いた。「……手順があることは、確認できました。発動前に定められた順序で何かに呼びかけて、応答を得てから次に進む——その構造が確かにあると」

「答え、ということは」

「なぜそういう手順があるのか——というところまでは、今日は出ていません。構造があることを確認した段階です」アルは言った。「ただ——`mana_routing`との往復として見えたことで、それが何に向けた呼びかけなのかの輪郭は、少し出てきた気がします」

　フィンは黙って聞いていた。

「すべての術者が、発動のたびに同じ手順を踏んでいる。属性が違っても、場所が違っても——その手順は変わらない。ということは——」アルはゆっくり言った。「その手順は術者が個別に決めているのではなく、どこかで定められている、ということになる」

　フィンは少し考えてから言った。「どこかで定められた、規定された手順」

「そうです」

「それが`mana_routing`との間で成立している——ということは、`mana_routing`を作った側が、その規定を決めた、ということになる」

「……おそらく、そうなります」

　路地の角に、翡翠の水差し亭の看板が見えた。

　リーラが「今日は休め」と言った。「歩けてるのは分かるが、顔が動いてない」

「そうします」

「明日、飯は食えよ」

　アルは少し笑った。笑った、かどうかは自分では分からなかったが、「はい」と答えた。

　宿の扉を押した。

　頭の後ろの締まりは、まだあった。

　二階への階段を上りながら、今日の出力の語がもう一度頭の中に並んだ。

　`phase[0] carries: [attribute] + [intent] + [caster_id]`。

　`routing: received by mana_routing substrate`。

　`response: channel allocation returned per caster`。

　（呼び出しを受け取り、応答して、チャンネルを割り当てる——`mana_routing`はその処理をしている）

　封印区画の出力で見た `mana_routing`。`world_management_system` の配下に位置するサブモジュール。大崩壊後に再構築されたバージョン。

　（あれが——毎回の魔法発動で、この世界のすべての術者の「呼び出し」を処理している）

　前世の言葉では、それを「呼び出し規約」と言った。どのようにリクエストを送り、どのようにレスポンスを受け取るかを定めた、取り決めのことだった。どんな処理も、その規約に沿ってやり取りしなければ動かない。

　（この世界の魔法は——そういう規約に従って動いている）

　断言はできない。出力の解釈は間違いうる。

　ただ——今日確認したことは確かだった。

　発動前に手順がある。手順は定められている。その手順に従って、何かが応答してくる。

　（定められた呼び出し手順。それが、魔法の発動の始まりにある）

　部屋の扉を開けた。

　窓から初夏の光が入っていた。羊皮紙が机の端に積まれていたまま残っていた。

　アルはそれをそのままにして、椅子に座った。

　今日はもう書かない。頭が動ける状態ではなかった。

　記録は明日でいい。

　ただ——一点だけ、書き留めておきたかった。小さな羊皮紙の切れ端を手に取り、短く書いた。

　「発動前の手順——定められた呼び出し規約として観測済み。`mana_routing`の応答を確認。`phase[0]-[3]`の順序一定」。

　それだけ書いて、端に置いた。

　椅子の背に体を預けると、頭の後ろの締まりが少しだけ薄れた。

　窓の外の石畳に、夕方の光が来始めていた。
